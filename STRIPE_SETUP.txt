                            returns a Stripe-hosted onboarding URL

                            routes funds to the seller's account minus 3% platform fee

                            and syncs it back to Supabase

HOW IT ALL WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

SELLER ONBOARDING:
  1. Seller logs into McBook dashboard → clicks "Payouts" in sidebar
  2. Fills in name, DOB, address, BSB + account number
  3. McBook calls create-connect-account → Stripe Express account created
  4. Seller redirected to Stripe-hosted page to verify their identity (photo ID etc.)
  5. Stripe fires account.updated webhook → McBook updates seller status to 'enabled'
  6. Seller redirected back to stripe-return.html → sees "Payouts enabled ✓"
  7. Bank details are NEVER stored in McBook — Stripe holds everything

CUSTOMER PAYMENT:
  1. Customer books a service via the widget
  2. Widget calls create-payment-intent with the booking + seller details
  3. Stripe creates a PaymentIntent — full amount charged to customer
  4. Stripe keeps 3% as McBook's platform fee, transfers rest to seller
  5. Stripe fires payment_intent.succeeded → McBook marks booking as paid
  6. Stripe automatically pays seller's bank account daily

REFUNDS:
  1. Refund issued via Stripe Dashboard or API
  2. Stripe fires charge.refunded webhook
  3. McBook marks booking as cancelled + refunded in database

SELLER PAYOUTS:
  1. Happen automatically — no action needed
  2. Daily schedule set when account was created
  3. Stripe fires payout.paid webhook → logged in payments table

5. Update STRIPE_WEBHOOK_SECRET + STRIPE_CONNECT_SECRET with the new whsec_ values
6. Redeploy functions: supabase functions deploy (all four)
